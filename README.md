# Командная строка
<br>

|Команда|Описание|Комментарий|
|-|-|-|
|**history**|Показать историю введённых команд|
|**clear**|Очистить экран терминала|
|**man** имя\_команды|Показать руководство по команде|
|**!!**|Повторить последнюю введённую команду|
|**pwd**|Показать рабочую папку|Print working directory. <br> Показывает путь к папке, где мы сейчас находимся. По умолчанию — домашняя директория|
|**ls**|Отобразить файлы и папки в текущей директории|List directory contents|
|**l -a**|Отобразить файлы и папки в текущей директории, в том числе скрытые||
|**cd** имя\_папки|Переместиться из одной папки в другую|Change directory. <br> Например, `cd Desktop`. <br> Если в названии папки есть пробелы, используйте кавычки: cd "Рабочий стол"|
|**cd** имя\_папки/имя\_вложенной\_папки|Перейти к папке внутри папки|Например, `cd first-project/test` <br> # Переход к папке test внутри папки first-project|
|**cd ..**|Перейти к папке на уровень выше||
|**cd \~**|Перейти в домашнюю папку||
|**mkdir** имя\_папки|Создать новую папку|Команде нужно передать имя новой папки. <br> Например, `mkdir dev`|
|**touch** имя\_файла.расширение|Создать новый файл|Например, `touch hello-world.txt`. <br> Можно создать несколько файлов через пробел: <br> `touch hello.txt hello2.txt hello3.txt`|
|**mv** старое\_имя новое\_имя|Переименовать файл|Move. <br> Например, `mv hello-world.txt HelloWorld.txt`|
|**mv** имя\_файла имя\_папки|Переместить файл в другую папку|Например, `mv HelloWorld.txt temp`|
|**cp** имя\_файла имя\_другого\_файла|Копировать файлы и папки|Например, `cp helloPracticum.txt practicum.txt` <br> # В этом примере информация из источника helloPracticum.txt скопируется в назначение practicum.txt|
|**rm** имя\_файла|Удалить файл|Remove. <br> Например, `rm bad-emotions.txt`|
|**rmdir** имя\_папки|Удалить пустую папку|Если в папке есть какие-то файлы, командная строка не удалит её|
|**rm -r** имя\_папки|Удалить папку со всем содержимым|Мы вызвали команду `rm` с ключом `r`. <br> Рекурсивно удалит всё содержимое директории безвозвратно (мимо корзины)|
|**rm -rf** ~/Library/Developer/Xcode/DerivedData|Удалить временные файлы Xcode|Папка `DerivedData` обычно содержит временные файлы, созданные Xcode в процессе сборки и запуска проектов: скомпилированные бинарные файлы, индексы для ускорения поиска и кэшированные данные. Удаление этой папки может быть полезным, чтобы освободить место на диске или решить некоторые проблем, связанные с Xcode. При следующем запуске Xcode воссоздаст необходимые файлы в этой папке|
|**echo** "текст"|Вывод текста на экран|Сама команда не записывает данные в файл, она их лишь выводит. <br> Например, `echo "Hello, practicum"`|
|**echo** "текст" **>>** имя\_файла|Запись текста в файл|Символ перенаправления вывода >> . <br> Например, `echo "Hello, practicum" >> practicum.txt` <br> # Запишет строку “Hello, practicum” в файл|
|команда **>>** имя\_файла|Запись вывода команды в файл|Например, `pwd >> current_dir.txt` <br> # Запишет в файл current_dir.txt текущую директорию|
|**cat** имя\_файла|Вывод содержимого файла в командную строку|Например, `cat output.txt`|
|**head** имя\_файла|Показать первые строки файла||
|**tail** имя\_файла|Показать последние строки файла||

# Git команды
<br>

|Команда|Описание|Комментарий|
|-|-|-|
|**git --version**|Узнать установленную версию Git|Git идёт в специальном пакете программ для Xcode — Command Line Tools. Вероятно, когда вы настраивали Xcode — этот пакет уже установился|
|**xcode-select --install**|Установить Git принудительно||
|**git config --global user.name** "имя\_пользователя"|Указать имя пользователя|Вводится латиницей в кавычках|
|**git config --global user.email** электронная\_почта|Указать электронную почту||
|**git config --list**|Список всех свойств конфига||
|**git config --global core.ignorecase false**|Выкл. нечувствительность к регистру||
|**git init**|Создать репозиторий из текущей папки|Внутри папки проекта появится новая папка `.git`, в ней хранится вся служебная информация: <br><br> `HEAD` — текстовый файл с указанием, куда ведёт ссылка HEAD. <br> `config` — текстовый файл с настройками репозитория. <br> `description` — текстовый файл с описанием репозитория. <br> `hooks` — папка с хуками Git. <br> `info` — папка с дополнительной информации о репозитории. <br> `objects` — папка с объектами Git. <br> `refs` — папка со ссылками|
|**rm -rf .git**|«Разгитить» папку|Удаляем скрытую подпапку `.git`. <br> Вся история проекта будет стёрта без возможности восстановления — останется только последняя версия файлов|
|**git status**|Узнать текущее состояние репозитория||
|**git add** имя\_файла|Добавить файл для сохранения в новой версии|Например, `git add hello.txt`|
|**git add --all**|Добавить несколько файлов и подготовить их к сохранению||
|**git add .**|Добавить всю текущую папку для сохранения|Альтернатива ключу `--all`|
|**git commit -m** "текст"|Сделать коммит и присвоить ему название|Например, `git commit -m "Мой первый коммит"`|
|**git remote add** origin git&#64;github.com:\<ВАШ НИКНЕЙМ\>/first-project-git.git|Добавить ссылку на удалённый репозиторий|Команда связывает локальный репозиторий с удалённым — или, проще говоря, сохраняет ссылку на страницу удалённого репозитория: <br><br> `git remote add` — команда для добавления нового удалённого репозитория. <br><br> `origin` — короткое имя, которое мы присваиваем удалённому репозиторию. Origin — стандартное название для основного удалённого репозитория, но вы можете использовать любое другое. <br><br> `git@github.com:<ВАШ НИКНЕЙМ>/first-project-git.git` — адрес удалённого репозитория на GitHub для SSH-подключения. Здесь \<ВАШ НИКНЕЙМ\> лучше заменить на реальный никнейм на GitHub, first-project-git — на название вашего репозитория на GitHub|
|**git branch -M** main|Переименовать ветку (в main)|Ключ `-M` отвечает за переименование, а `main` — это название самой ветки. <br><br> Это действие обычно делается один раз, чтобы установить стандартное название для вашей основной ветки в проекте|
|**git push -u** origin main|Запушить изменения|Отправляет изменения из локальной ветки `main` в главную ветку удалённого репозитория (`origin`) и установливает её как основную для будущих использований команды `push`. После первого опубликованного коммита в ветку можно пользоваться короткой версией команды и писать просто `git push`. <br><br> `git push` — указывает Git отправить (или «запушить») изменения из вашего локального репозитория в удалённый. <br><br> `-u` – этот ключ означает set upstream: мы устанавливаем связь между локальной веткой и удалённой. В следующий раз, когда вы будете отправлять или получать изменения, Git будет «знать», откуда брать или куда отправлять данные. <br><br> `origin` — название стандартного удалённого репозитория. <br><br> `main` — название ветки, которую вы хотите отправить в удалённый репозиторий |
|**git clone** ссылка\_на\_проект название\_папки|Скачать проект в указанное место|С помощью этой команды можно скачать удалённый репозиторий на локальный компьютер. Например: <br><br> `git clone git@github.com:<ВАШ НИКНЕЙМ>/first-project-git.git second-project-git` <br><br> `git clone` — так объявляется команда Git для клонирования репозитория. <br><br> `git@github.com:<ВАШ НИКНЕЙМ>/first-project-git.git` — адрес репозитория в GitHub, с которого вы хотите сделать копию. <br><br> `second-project-git` — название папки на вашем компьютере, куда будет склонирован репозиторий (создастся с указанным названием, если такой нет). Если вы не укажете это название, Git создаст папку с тем же названием, что и у репозитория на GitHub|
|**git pull**|Скачать проект и объединить с текущей рабочей версией|Команда выполняет две основные операции: <br><br> `git fetch`, которая скачивает изменения с удалённого репозитория, <br> `git merge`, которая объединяет эти изменения с вашей текущей рабочей версией. <br><br> 1. Перед выполнением `git pull` убедитесь, что ваша рабочая версия чиста, то есть все изменения зафиксированы: сделан commit. <br> 2. Иногда после выполнения `git pull` могут возникнуть конфликты слияния, если одни и те же части кода были изменены локально и в удалённом репозитории. В таком случае Git попросит вас разрешить эти конфликты перед тем как продолжить|
|**git branch**|Показать ветки, которые есть в проекте|При вызове команды выводятся ветки, которые есть в проекте. Звёздочкой (\*) отмечено, в какой ветке вы находитесь сейчас. Например: <br> `another_branch` <br> `* main` <br> `new_branch`|
|**git branch** название\_ветки|Создать новую ветку|Например, `git branch new_branch`. <br> При этом мы остаёмся в старой ветке|
|**git checkout** название\_ветки|Переключиться между ветками||
|**git checkout -b** название\_ветки|Создать новую ветку и сразу на неё переключиться|Эта команда делает то же самое, что и последовательность команд: <br><br> `git branch название_ветки && git checkout название_ветки`<br><br> Например, `git checkout -b another_branch` |
|**git merge** название\_ветки|Слить ветки|Перед тем как начать слияние, нужно перейти в ветку, куда должны добавиться изменения. Обычно это главная ветка. Перейдите в неё и вызовите команду `git merge` с именем присоединяемой ветки `new_branch` в качестве параметра. Все коммиты из `new_branch` будут добавлены в главную ветку:<br><br> `git checkout main` # Переключились на главную ветку <br> `git merge new_branch` # Объединили ветки|

# Горячие клавиши
<br>

|Комбинация|Описание|Комментарий|
|-|-|-|
|**Ctrl + c**|Отменить ввод команды||
|**Ctrl + a**|Переместить курсор в начало строки||
|**Ctrl + e**|Переместить курсор в конец строки||
|**Tab**|Автодополнение при вводе названий папок / файлов|Чтобы не вводить названия файлов и папок полностью, можно начать писать их имя и нажать клавишу `Tab`. Командная строка допишет путь сама, если соответствующий файл или папка есть в текущей директории|
